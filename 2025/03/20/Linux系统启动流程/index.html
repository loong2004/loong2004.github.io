Linux系统启动流程

Loong

 875 Words  3 Mins  

Linux系统的启动分为五个流程：

  1. 内核引导
  2. 运行init
  3. 系统初始化
  4. 建立终端
  5. 用户登录

一，内核引导

当计算机打开电源后，首先是BIOS开机自检，按照BIOS中设置的启动设备（通常是硬盘）来启动。

操作系统接管硬件以后，首先读入 /boot 目录下的内核文件。

二，运行init

init 进程是系统所有进程的起点，你可以把它比拟成系统所有进程的老祖宗，没有这个进程，系统中任何进程都不会启动。

init程序类型又分为以下三种：

  • SysV: init, CentOS 5之前, 配置文件： /etc/inittab。

  • Upstart: init, CentOS 6, 配置文件： /etc/inittab, /etc/init/*.conf。

  • Systemd： systemd, CentOS 7,配置文件： /usr/lib/systemd/system、 /etc/systemd/system。

1，如何验证自己系统的init程序类型？

  • 若输出 systemd，则系统使用 systemd。
  • 若输出 init，可能是 SysV init 或 Upstart（旧版本）。

2，运行级别

许多程序需要开机启动。它们在Windows叫做”服务”（service），在Linux就叫做”守护进程”（daemon）。

init进程的一大任务，就是去运行这些开机启动的程序。

但是，不同的场合需要启动不同的程序，比如用作服务器时，需要启动Apache，用作桌面就不需要。

Linux允许为不同的场合，分配不同的开机启动程序，这就叫做”运行级别”（runlevel）。也就是说，启动时根据”运行级别”，确定要运行哪些程序。

运行级别名称用途
0关机模式关闭所有服务并安全关机（init 0 或 poweroff）。
1单用户模式救援模式，仅启动最小服务（无网络、多用户），用于系统修复（init 1）。
2多用户模式（无网络）支持多用户登录，但不启用网络服务（部分旧发行版使用，现已少见）。
3多用户文本模式完整的多用户模式，启用网络服务，但无图形界面（服务器常用）。
4未定义保留级别，默认与级别 3 相同，可自定义用途。
5图形界面模式与级别 3 类似，但启用图形界面（桌面版默认级别）。
6重启模式关闭所有服务并重启系统（init 6 或 reboot）。
